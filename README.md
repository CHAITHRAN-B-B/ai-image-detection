AI: Multi-Agent Deepfake Verification Engine

Track: Agents for Good (or Freestyle)

📌 Problem Statement

As generative AI models become increasingly sophisticated, synthetic media is becoming indistinguishable from reality. Malicious actors utilize these tools to generate deepfakes, spreading misinformation, manipulating public opinion, and damaging reputations. Currently, verifying the authenticity of an image requires manual expert analysis or heavy, expensive cloud infrastructure.

We need an accessible, automated way to analyze images for AI artifacts to restore trust in digital media.

💡 The Solution

The Cyber-Forensics AI is an advanced, multi-agent diagnostic pipeline built using the core concepts from the Google AI Agents Intensive Vibe Coding Course.

It securely ingests suspect images and utilizes a specialized multi-agent workflow to isolate generative anomalies (such as structural blending, melted textures, and spatial inconsistencies). It then synthesizes these findings into a cryptographic visual report, drawing precise bounding boxes over the fraudulent areas. To make this accessible, the entire pipeline is wrapped in a high-tech, deployed Streamlit web application.

🏗️ Architecture & Course Concepts Applied

This project heavily leverages the tools and methodologies taught in the Intensive Vibe Coding course, specifically satisfying the following requirements:

1. Multi-Agent System (ADK Paradigm)

The core of the application is a decoupled, two-agent orchestration system:

Agent 1 (The Visual Inspector): Powered by gemini-2.5-flash, this agent acts as the primary detective. It is prompted with strict instructions to look for AI artifacts (e.g., lack of joints in fingers, clothing boundaries melting). Using the new google-genai SDK, we utilized types.GenerateContentConfig to enforce a strict application/json output, mapping the exact X/Y coordinates and radii of the anomalies.

Agent 2 (The Reporter): This agent synthesizes the JSON findings, triggering the execution of custom tools to visually highlight the threats.

2. Agent Skills (Custom Tools)

To bridge the gap between AI analysis and tangible outputs, I developed the annotate_image tool.

Implementation: A Python-based programmatic skill utilizing the Pillow library. It parses the coordinate array generated by Agent 1 and physically draws targeted red bounding boxes/ellipses directly onto the evidence image, providing undeniable visual proof of the manipulation.

3. Security Features (Pre-Processing Gateway)

To ensure the agent environment remains secure from malicious payloads, I built a strict validate_image_security tool that runs before any LLM processing occurs.

Implementation: The gateway enforces a strict 5MB payload limit to prevent Denial of Service (DoS) attacks. It verifies allowed MIME types (JPG/PNG) and utilizes PIL.Image.verify() to deeply inspect the cryptographic image headers, blocking corrupted or disguised executable files from entering the pipeline.

4. Deployability & UI

The entire agent pipeline was transformed from a local command-line script into a fully deployed web application using Streamlit Community Cloud. It features a custom CSS UI, asynchronous loading states, and a dynamic JavaScript-powered magnifying glass to allow users to inspect the microscopic artifacts identified by the agents.
